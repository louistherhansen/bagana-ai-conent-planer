## Recommended Workflow (Screen Recording)

### Step 1: Prepare Platform
1. Ensure dev server is running (`npm run dev`)
2. Test all features shown in video
3. Use demo data or create sample content
4. Clean browser (no extensions, bookmarks visible)
5. Set browser zoom to 100%

### Step 2: Record Screen
1. **Open OBS Studio** or Loom
2. Set recording area to 1920x1080
3. Record each section separately:
   - Chat interface demo
   - Dashboard walkthrough
   - Content plans view
   - Sentiment analysis
   - Trends view
4. Keep recordings smooth (no mouse jitter)
5. Record 5-10 seconds longer than needed

### Step 3: Record Voiceover
1. **Use Audacity** (free) or professional mic
2. Record in quiet environment
3. Read from `voiceover-script.txt`
4. Record multiple takes
5. Export as WAV or MP3 (high quality)

### Step 4: Edit Video
1. **Import:** Screen recordings + voiceover
2. **Sync:** Align voiceover with visuals
3. **Add:** Logo, text overlays, transitions
4. **Music:** Add background track (low volume)
5. **Export:** MP4, 1920x1080, 30fps

## Music Selection Tips

### Genre
- Upbeat, tech-forward
- Instrumental (no lyrics)
- Professional, modern

### Tempo
- Moderate pace (matching narration)
- Not too fast or slow
- Builds energy but doesn't distract

### Sources
- **YouTube Audio Library** - Free, no attribution needed
- **Free Music Archive** - Creative Commons
- **Epidemic Sound** - Subscription (if budget allows)

### Volume
- Keep at 20-30% of narration volume
- Fade in/out smoothly
- Don't compete with voiceover

## Text Overlay Best Practices

### Readability
- **Font:** Sans-serif, bold
- **Size:** Large enough to read on mobile
- **Color:** High contrast (white on dark, dark on light)
- **Position:** Safe area (not too close to edges)

### Timing
- Appear 0.5s before relevant narration
- Stay on screen 2-3 seconds minimum
- Fade in/out smoothly

### Content
- Key phrases only
- 3-5 words maximum per overlay
- Don't repeat narration word-for-word

## Export Settings

### For YouTube
- **Format:** MP4
- **Codec:** H.264
- **Resolution:** 1920x1080
- **Frame Rate:** 30fps
- **Bitrate:** 8-12 Mbps
- **Audio:** AAC, 48kHz, Stereo

### For LinkedIn
- **Format:** MP4
- **Resolution:** 1920x1080 or 1280x720
- **Max File Size:** 200MB
- **Duration:** Up to 10 minutes

### For Website Embedding
- **Format:** MP4 (with WebM fallback)
- **Resolution:** 1920x1080
- **Compression:** Optimize for web (smaller file)

## Common Mistakes to Avoid

1. ❌ **Too fast pacing** - Give viewers time to absorb
2. ❌ **Low audio quality** - Use good microphone
3. ❌ **Cluttered visuals** - Keep it clean and focused
4. ❌ **Inconsistent branding** - Use brand colors/fonts
5. ❌ **No clear CTA** - Always end with action
6. ❌ **Poor lighting** - If using webcam, ensure good light
7. ❌ **Background noise** - Record in quiet space
8. ❌ **Too much text** - Keep overlays minimal

---

- [ ] Text overlays are readable
